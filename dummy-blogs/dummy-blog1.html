Building to-do app

24 September

While I started with this project I found little difficult to develop logic to build this plan. Then it became easier for me when I decided I should break it into parts and should try to develop it by breaking it into various parts. So I first developed input for entering todo and a button for adding todo. Then made all the lists using ul tag and put li tag in it and put all the items on the list I wanted. Then imported useState from react and made an useState function. Then made an array of inputs and todos. Then the plan was to make an add button to make an option to add items by the user. so I made an add button to make it run I made a function addTodo which contained an array of input that the user will enter and...todos which will bring all the list of todos. Also added edit button so the user can also edit the items in the list.

Then I made showTodo function that renders all elements with add and edit button. On clicking the edit button, editButton function will run.

Then for the deletion of tasks user might also need a deleting feature so added a del button. For implementing this I made delTodos function with filter method in it.

Then made a function handleInputChange so that if a user puts any changes in the input it should get saved. To prevent addition of todos with empty title, I added an if condition that checks if the input is not equal to an empty string then only the further code should run.

Then in the editTodo function if the user is not changing any name keeping it empty it should not change anything so for that I added an if condition that if the input is not equal to null then only the further process the map method.
